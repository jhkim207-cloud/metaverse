HK G&Tech 생산관리 시스템

유리 가공 생산 전 과정을 관리하는 통합 시스템입니다.
주문부터 출고까지의 워크플로우와 각 도메인의 관계를 보여줍니다.

핵심 흐름 (워크플로우)
FK 관계
참조 관계 (코드)
1:M 1대다
M:N 다대다

도메인 상세 설명

📋
마스터데이터
시스템의 기준 정보를 관리합니다. 거래처(고객/공급사), 시공 현장, 유리/부자재 품목, 작업자 정보 등 모든 업무의 기초가 되는 데이터입니다.
business_partner site_master site_price item_master worker worker_assignment_type worker_daily_assignment code_master standard_terms cutting_daily_report
거래처 → 현장 : 1:M (하나의 거래처에 여러 현장)
작업자 → 일일배치 : 1:M (작업자별 날짜마다 배치)
거래처 → 주문 : 1:M (customer_cd로 참조)
📝
영업 / 주문
고객으로부터 받은 유리 가공 주문을 등록하고 관리합니다. 프로젝트(건축 현장) 주문와 임가공(단발성) 주문로 구분되며, 주문 확정 후 생산계획으로 이어집니다.
sales_order_header sales_order_detail sales_order_upload
주문헤더 → 주문상세 : 1:M (하나의 주문에 여러 품목 라인)
주문 → 생산계획 : M:N (plan_order_detail 매핑 테이블)
주문 → 출고 : 1:M (주문번호로 출고 연결)
🏭
생산
주문를 기반으로 생산계획을 수립하고, 작업지시를 내려 실제 생산을 진행합니다. 복층1호기/2호기, 재단/강화 라인에 작업을 배정하고, 생산 완료 후 실적을 기록합니다. 완제품은 포장 공정을 거쳐 출고 준비됩니다.
production_plan production_plan_detail production_plan_order_detail work_request production_result subcontract_order packing_order
생산계획 → 계획상세 : 1:M (FK plan_id)
계획상세 → 작업지시 : 1:M (계획상세 기준으로 작업지시 생성)
작업지시 → 생산실적 : 1:M (work_request_no 참조)
생산계획 ↔ 주문상세 : M:N (plan_order_detail)
📦
구매 / 발주
생산에 필요한 원자재(유리 원판, 부자재)를 공급사에 발주합니다. 발주 후 입고(goods_receipt)가 완료되면 재고에 반영됩니다.
purchase_order purchase_order_detail goods_receipt
발주 → 발주상세 : 1:M (po_no로 연결)
발주 → 입고 : 1:M (po_no 참조)
주문 → 발주 : 1:M (주문에 따른 자재 발주)
📥
재고 / 자재
유리 원판과 부자재의 현재고, 가용수량, 예약수량을 실시간 관리합니다. 입고/출고/이동/조정 이력을 모두 추적하며, 최소/최대 재고 기준으로 경고합니다. 유리 운반용 용기(컨테이너)의 위치와 재고도 관리합니다.
inventory inventory_transaction goods_receipt container_inventory
재고 → 입출고이력 : 1:M (material_cd 기준)
발주 → 입고 → 재고 : 입고 시 재고 증가
출고 → 재고 : 출고 시 재고 차감
🚚
출고 / 물류
포장 완료된 제품을 현장으로 출고합니다. 출고 지시서를 생성하고, 운송사/차량 배정 후 실출고를 처리합니다. 주문번호와 현장코드로 어떤 현장에 배송되는지 추적합니다.
delivery_header delivery_detail
출고헤더 → 출고상세 : 1:M (delivery_no로 연결)
주문 → 출고 : 1:M (order_no 참조)
출고 → 재고 : 출고 시 재고 차감
⚙️
시스템 / 공통
사용자 계정, 부서/조직, 역할 및 메뉴 권한을 관리합니다. 모든 사용자 활동은 감사 로그에 기록되며, 역할 그룹을 통해 권한을 상속받을 수 있습니다.
users roles permissions organizations menus role_menus user_roles role_groups role_group_members role_group_inheritance audit_logs
사용자 ↔ 역할 : M:N (user_roles 매핑)
역할 ↔ 메뉴 : M:N (role_menus 매핑)
조직 → 사용자 : 1:M (organization_id)
💬
생산 회의
매일 아침 진행하는 생산 회의를 기록합니다. 작업지시 현황 리뷰, 이슈 논의, 액션아이템 할당 등을 체계적으로 관리하며, 미완료 항목은 다음 회의로 이월됩니다.
daily_production_meeting daily_meeting_note daily_meeting_action_item
회의 → 메모 : 1:M (meeting_id)
회의 → 액션아이템 : 1:M (meeting_id)
액션아이템 → 작업지시 : N:1 (work_request_no 참조)
