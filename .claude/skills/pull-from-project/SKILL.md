# /pull-from-project - 프로젝트 결과 표준 환류 Skill

## 개요

완료된 프로젝트에서 표준화 후보를 추출하여 표준 템플릿에 반영하는 Skill입니다.

**사용 대상**: 표준 템플릿 (project_temp, project_temp_manufacturing 등)

## 사용법

```
/pull-from-project [프로젝트 경로]
/pull-from-project "../project_samsung"
/pull-from-project "C:/projects/project_samsung"
```

## 입출력

- **입력**: 대상 프로젝트 경로
- **산출물**: `docs/standard-candidates/{프로젝트명}-candidates.md`

---

## 실행 체크리스트

```
□ 1. 대상 프로젝트 경로 확인
□ 2. 프로젝트의 prd/, assets/, db_dic/ 스캔
□ 3. 현재 표준 대비 신규/변경 항목 추출
□ 4. 표준화 후보 리포트 생성
□ 5. 컨설턴트에게 검토 요청
□ 6. 승인된 항목만 표준에 반영
```

---

## 분석 대상

| 디렉토리 | 분석 항목 |
|----------|----------|
| `prd/` | 신규 도메인, 새 User Story, 새 기능 요구사항 |
| `prd/*/0x-*.md` | 확장된 API, 새 화면, 추가 테스트 시나리오 |
| `assets/domain/` | 새 도메인 자산 정의 |
| `db_dic/sql/` | 새 테이블, 새 컬럼 |
| `db_dic/dictionary/` | 새 표준 용어 |

---

## 검증 항목 (5개)

| # | 항목 | 유형 | 기준 |
|---|------|------|------|
| 1 | 프로젝트 경로 유효성 | 자동 | 경로 존재 확인 |
| 2 | 신규 항목 추출 | 자동 | prd/, assets/, db_dic/ 비교 |
| 3 | 변경 항목 추출 | 자동 | 파일 diff 분석 |
| 4 | 후보 분류 | 수동 | 도메인별, 유형별 분류 |
| 5 | 컨설턴트 검토 요청 | 수동 | 승인/반려 선택 UI 제공 |

```markdown
## 승인 옵션

[A] 전체 승인 - 모든 후보를 표준에 반영
[S] 개별 선택 - 체크된 항목만 반영
[R] 전체 반려 - 반영 안 함
```
