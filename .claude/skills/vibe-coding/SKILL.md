# Vibe Coding 가이드 스킬

## 개요

AI 보조 개발(Vibe Coding) 시 안전하고 효과적인 코드 생성을 위한 가이드입니다.

## Vibe Coding 워크플로우

```
┌─────────────────────────────────────────────────┐
│  1. Ideation                                    │
│  └─ 자연어로 기능 설명                          │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  2. Generation                                  │
│  └─ AI가 초기 코드 생성                         │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  3. Review & Understand                         │
│  └─ 개발자가 코드 검토, 이해, 수정              │
│  └─ 🚨 이 단계를 건너뛰면 "Vibe Coding 리스크"  │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  4. Test & Validate                             │
│  └─ 단위/통합 테스트 실행                       │
│  └─ 보안 스캔 실행 → /security 스킬 참조        │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  5. Document & Commit                           │
│  └─ 의미 있는 커밋 메시지                       │
│  └─ 필요시 ADR 작성                             │
└─────────────────────────────────────────────────┘
```

## AI 활용 영역 분류

| 영역 | AI 활용도 | 검토 수준 |
|------|---------|---------|
| 보일러플레이트 코드 | 높음 | 간단 검토 |
| CRUD 로직 | 높음 | 테스트 필수 |
| 비즈니스 로직 | 보통 | 상세 검토 |
| 보안 관련 코드 | 낮음 | 전문가 검토 |
| 아키텍처 결정 | 낮음 | ADR 필수 |

## AI 코드 생성 후 필수 체크리스트

```markdown
1. [ ] 생성된 코드를 라인별로 이해했는가?
2. [ ] 단위 테스트를 작성/실행했는가?
3. [ ] 보안 취약점이 없는가? → /security 스킬 참조
4. [ ] 기존 코드 패턴과 일관성이 있는가?
5. [ ] 불필요한 의존성이 추가되지 않았는가?
6. [ ] 에러 처리가 적절한가?
```

## 효과적인 프롬프트 구조

```
[Context] 프로젝트 배경과 기술 스택
[Task] 구체적인 작업 내용
[Constraints] 제약 조건과 규칙
[Examples] 기존 코드 패턴 예시
[Output] 원하는 출력 형식
```

### 예시

```
[Context]
Spring Boot 3.x + MyBatis 프로젝트입니다.
기존 UserService 패턴을 따릅니다.

[Task]
주문(Order) 도메인의 CRUD API를 생성해주세요.

[Constraints]
- Service 클래스는 300줄 이하
- 모든 public 메서드에 @Transactional
- NotFoundException 사용

[Examples]
UserService.java 참고

[Output]
Controller, Service, Mapper, DTO 파일
```

## AI에게 요청하지 말아야 할 것

- 민감한 비즈니스 로직의 자동 생성
- 보안 인증/암호화 코드
- 금융 계산 로직
- 법적 컴플라이언스 관련 코드

## 보안 관련

> **보안 상세 가이드**: `/security` 스킬 참조
>
> AI 생성 코드는 보안 취약점이 포함될 수 있습니다.
> OWASP Top 10, 입력값 검증, JWT 설정 등은 `/security` 스킬을 확인하세요.
