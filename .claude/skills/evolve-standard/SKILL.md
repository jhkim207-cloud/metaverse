# /evolve-standard - 표준 자체 개선 Skill

## 개요

외부 Best Practice를 조사하여 현재 표준을 개선하는 Skill입니다.

**사용 대상**: 표준 템플릿 (project_temp, project_temp_manufacturing 등)

## 사용법

```
/evolve-standard [도메인명]
/evolve-standard "재고관리"
/evolve-standard "order-management"
/evolve-standard --all  # 전체 도메인 분석
```

## 입출력

- **입력**: 개선 대상 도메인명 (또는 --all)
- **산출물**: `docs/standard-evolution/{도메인명}-evolution.md`

---

## 실행 체크리스트

```
□ 1. 대상 도메인 현재 표준 분석
□ 2. 외부 Best Practice 조사 (멀티 LLM)
□ 3. Gap 분석 수행
□ 4. 표준 개선 제안 리포트 생성
□ 5. 컨설턴트에게 검토 요청
□ 6. 승인된 항목만 표준에 반영
```

## 4. 개선 제안

### 4.1 단기 (1-3개월)

#### 🔴 납기 자동 산출 (ATP 단순 버전)
- [ ] **기능**: 재고 기반 납기 자동 계산
- **구현 방안**:
  - 가용 재고 확인 API 추가
  - 납기 계산 로직 (재고 ÷ 일평균 출하량)
- **예상 효과**: 납기 입력 시간 80% 감소
- **필요 변경**:
  - API: GET /inventory/available/{productId}
  - API: POST /orders/calculate-delivery-date
  - 컬럼: orders.calculated_delivery_date

### 4.2 중기 (3-6개월)

#### 🔴 여신 한도 관리
- [ ] **기능**: 주문 시 고객 여신 자동 검사
- **구현 방안**:
  - 고객별 여신 한도 마스터
  - 미수금 + 신규주문 합계 vs 한도 비교
- **필요 변경**:
  - 테이블: customer_credit_limits
  - API: GET /customers/{id}/credit-status

### 4.3 장기 (6개월+)

#### 🟢 주문 분할/통합
- [ ] **기능**: 다중 창고 기반 주문 자동 분할
- **검토 필요**: 현재 단일 창고 전제, 다중 창고 필요성 확인

## 5. 멀티 LLM 분석 결과

### GPT-5.2 (기술 관점)
- ATP 구현 시 캐싱 전략 필요 (재고 조회 빈도 높음)
- 여신 검사는 트랜잭션 내 동기 처리 권장

### Gemini-3-Pro (비즈니스 관점)
- 납기 자동화는 고객 만족도 직결
- 여신 관리는 매출 채권 리스크 감소 효과
