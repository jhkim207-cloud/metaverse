## Provisional 승인 프로세스

### 승인 옵션

```
[A] 전체 승인 (표준 문서에 추가)
    → UI_STANDARD.md / standards.json 업데이트

[P] 프로젝트 한정 (이번만 사용)
    → PRD에만 기록, 표준 문서 미변경

[S] 개별 선택
    → 항목별로 A/P/R 선택

[R] 전체 거부
    → 표준 항목으로 대체 필요
```

### 승인 후 자동 업데이트 규칙

**[A] 승인 시 표준 문서 자동 업데이트:**

#### UI 컴포넌트 승인 시
1. `docs/ui-standards/UI_STANDARD.md`의 적절한 섹션에 추가
2. 포함 내용:
   - 컴포넌트 설명
   - 코드 예시
   - Props 테이블
   - 접근성 가이드

#### DB 용어 승인 시
1. `db_dic/dictionary/standards.json`에 추가
2. 적절한 카테고리 선택 (새 카테고리 생성 가능)
3. 필수 필드:
   ```json
   "{column_name}": {
     "korean": "한글명",
     "english": "English Name",
     "type": "데이터타입",
     "constraint": "제약조건",
     "desc": "설명"
   }
   ```

#### 업데이트 확인 메시지

## 80% 완성도 판정

### 계산 방법

```
완성도 = (통과 항목 수 / 총 항목 수) × 100

설계 단계 총 항목 수: 46개 (Gate 0~7)
80% 기준: 37개 이상 통과

참고: Gate 8 (구현)은 /implement에서 별도 8개 항목으로 검증
전체 워크플로우 총계: 54개
```

### 판정 결과

| 완성도 | 판정 | 다음 단계 |
|--------|------|----------|
| ≥ 80% (37+) | 통과 | `/implement` 실행 가능 |
| < 80% | 미달 | 실패 Gate 재실행 필요 |

### 미달 시 안내 메시지

```
## 80% 완성도 미달

현재 완성도: XX% (XX/46)
부족 항목: XX개

### 실패한 Gate
- Gate X: {실패 항목 목록}

### 권장 조치
1. `/skill-name --fix` 로 해당 Gate 재실행
2. 수동 검토 항목 확인
3. `/prd-finalize` 재실행
```
