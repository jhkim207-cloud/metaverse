# 도구 사용 규칙

## 1. 코드베이스 검색 도구 선택

### 검색 범위에 따른 도구 선택

| 검색 범위 | 권장 도구 | 이유 |
|----------|----------|------|
| **넓은 범위** (프로젝트 전체, 여러 폴더) | **Explore 에이전트** | 종합적 탐색, 누락 방지 |
| **특정 폴더** (1~3개 디렉토리) | Grep | 빠른 패턴 매칭 |
| **특정 파일** (경로를 아는 경우) | Read | 직접 접근 |
| **파일명 패턴** | Glob | 와일드카드 검색 |

### 검색 유형에 따른 도구 선택

| 검색 유형 | 권장 도구 | 예시 |
|----------|----------|------|
| **텍스트/문자열 검색** | Grep 또는 Explore | "design-system" 참조 찾기 |
| **심볼/코드 탐색** | Code Intelligence (자동) | 함수 정의, 참조 추적 |
| **파일 구조 파악** | Explore 에이전트 | "인증 관련 파일 어디?" |
| **정규식 패턴** | Grep | `import.*from.*utils` |

### 의사결정 흐름

```
검색 필요
    │
    ├─ 검색 범위를 알고 있나?
    │   │
    │   ├─ YES: 특정 폴더 1~3개 → Grep
    │   │
    │   └─ NO: 프로젝트 전체 또는 불확실 → Explore 에이전트
    │
    └─ 파일 경로를 알고 있나?
        │
        ├─ YES: 정확한 경로 → Read
        │
        └─ 파일명 패턴만 → Glob
```

## 2. 금지 행동

1. **Grep 타임아웃 발생 시** → 같은 방식 재시도 금지, Explore 에이전트로 전환
2. **넓은 범위 검색에 Grep만 반복 사용** → Explore 에이전트 우선 사용
3. **Code Intelligence 명시적 호출 시도** → 자동 작동하므로 직접 호출 불가

---

## 3. 도구별 특성

### Grep
- **장점**: 빠름, 정규식 지원, 정확한 패턴 매칭
- **단점**: 경로 지정 필요, 타임아웃 가능
- **사용 시점**: 검색 범위가 명확할 때

### Explore 에이전트
- **장점**: 종합적 탐색, 누락 방지, 컨텍스트 효율
- **단점**: 상대적으로 느림
- **사용 시점**: 넓은 범위, 불확실한 검색

### Glob
- **장점**: 파일명 기반 빠른 검색
- **단점**: 파일 내용 검색 불가
- **사용 시점**: 파일 위치/존재 확인

### Code Intelligence (LSP)
- **작동 방식**: 자동 (명시적 호출 불가)
- **기능**: 심볼 정의, 참조 추적, 타입 정보
- **참고**: typescript-lsp, pyright-lsp 플러그인 필요
